运维工作流
节点安装 / 上报 / 路由
新增节点
创建节点记录后生成一键安装命令，部署到 Linux 节点服务器。
新增路由
为游戏名称和 ID 绑定节点及目标 UDP 地址，客户端 connect-intent 会按优先级调度。
升级控制面
当前选择
等待节点选择
在节点管理里选择节点后，可以在详情中生成安装命令和调整状态。
新增节点
创建节点记录后，再生成一键安装命令部署到 Linux 服务器。
新增路由规则
把游戏名称、游戏 ID、节点和上游目标地址绑定起来，客户端 connect-intent 会按优先级选择可用节点。
选择后会自动填入游戏 ID 和名称；如果游戏还不存在，请先到游戏管理新增。
只会列出当前控制面已经创建的节点。
数字越小越优先，同一游戏可以配置多个节点做备选。
新增游戏
维护游戏名称、平台和分类，路由规则会从游戏库里选择对应游戏。
业务后台和客户端 connect-intent 使用的游戏编号。
